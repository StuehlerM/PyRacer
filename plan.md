# Project Plan: 2D Reinforcement Learning Racing Game

## Overview

Build a 2D racing game where an AI agent learns to drive on autogenerated tracks using PyTorch reinforcement learning. The system consists of:

1. **Game Engine**: PyGame-based 2D racing game with procedural track generation
2. **RL Agent**: DQN-based agent that learns to drive using sensor inputs
3. **Training Pipeline**: Complete training and evaluation framework

## Phase 3: Reinforcement Learning ✅ COMPLETED

### Neural Networks (`rl/model.py`)
- **DQN**: Standard Deep Q-Network (3 hidden layers, ReLU activation)
- **Dueling DQN**: Separates value and advantage streams
- **ConvDQN**: Convolutional version for image inputs (optional)

### Memory Buffer (`rl/memory.py`)
- `ReplayBuffer`: Standard experience replay with random sampling
- `PrioritizedReplayBuffer`: Prioritized sampling based on TD error
- Serialization of states for storage

### RL Agent (`rl/agent.py`)
- **DQNAgent**: Double DQN implementation with:
  - Epsilon-greedy exploration
  - Experience replay
  - Target network (periodic updates)
  - Gradient clipping
  - Model save/load
- **RandomAgent**: Baseline agent for comparison

### Environment Wrapper (`rl/environment.py`)
- `RacingEnv`: Gym-like interface for single track
- `MultiTrackEnv`: Cycles through multiple tracks for generalization
- State normalization
- Statistics tracking (lap times, rewards)
- Rendering support

- Lap Complete: +100
- Checkpoint: +10
- Progress: +1 per %
- Collision: -10
- Time Penalty: -0.05 per step

## Implementation Notes

### Design Decisions

1. **DQN over PPO**: Chose DQN for simplicity and discrete action space. Can be extended to PPO later.

2. **Sensor-based State**: More generalizable than absolute coordinates. Similar to how humans perceive driving.

3. **Discrete Actions**: Simpler for DQN. Actions are: Accelerate, Partial Throttle, Brake, Left, Right.

4. **Progress-based Reward**: Encourages the agent to move forward along the track, not just survive.

5. **Target Network**: Critical for stable DQN training. Updated periodically (every 10 steps).

6. **Double DQN**: Reduces overestimation bias in Q-values.

### Potential Issues & Solutions

1. **Agent stuck in local optimum**: Try increasing exploration or using different track configurations.

2. **Slow learning**: Increase batch size, adjust learning rate, or try Dueling DQN.

3. **Track too difficult**: Reduce track complexity or increase track width.

4. **Car physics issues**: Adjust car parameters (speed, acceleration, friction) in config.
